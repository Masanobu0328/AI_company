# エージェント向け ナレッジベース操作ガイド

## 概要
このガイドは全エージェントがナレッジベースを操作する際の標準手順を定義する。

---

## 1. ノートを読む（READ）

### 手順
1. 対象フォルダを確認する（ideas / research / clients / projects）
2. ファイル名または内容で目的のノートを検索する
3. Frontmatterのタグ・日付でフィルタリングする

### 検索パターン
```
# クライアント情報を探す場合
knowledge_base/clients/[企業名]/

# 特定テーマのリサーチを探す場合
knowledge_base/research/ → タグ #業界名 で検索

# 進行中プロジェクトを確認する場合
knowledge_base/projects/ → status: active のノートを参照
```

---

## 2. ノートを更新する（UPDATE）

### 手順
1. 対象ノートを開く
2. Frontmatterの `更新日` を今日の日付に変更する
3. 変更内容を追記する（既存内容は削除しない）
4. 更新履歴セクションに変更概要を1行追記する

### 更新時の注意
- 既存の情報を上書きしない — 追記する形で残す
- 重要な変更は `## 更新履歴` セクションに記録する
- 関連ノートのリンクを確認・更新する

---

## 3. 新規ノートを作成する（CREATE）

### 手順
1. 適切なフォルダを選ぶ
2. 命名規則に従ってファイル名を決める
3. 対応するテンプレート（`_templates/`）をコピーする
4. Frontmatterを埋める
5. 関連ノートからリンクを張る

### フォルダ判断基準
```
アイデア・着想 → ideas/
市場・業界・競合調査 → research/
クライアント情報 → clients/[企業名]/
プロジェクト管理 → projects/[プロジェクト名]/
```

---

## 4. Frontmatterの標準形式

すべてのノートはファイル先頭に以下を記述する：

```yaml
---
title: [ノートタイトル]
作成日: YYYY-MM-DD
更新日: YYYY-MM-DD
作成者: [エージェント名]
tags: [タグ1, タグ2]
status: [active / archived / draft]
links: []
---
```

---

## 5. タグ一覧（標準タグ）

| タグ | 用途 |
|---|---|
| #クライアント | クライアント関連ノート |
| #リサーチ | 調査・分析ノート |
| #アイデア | アイデア・着想 |
| #プロジェクト | プロジェクト管理ノート |
| #営業 | 営業活動関連 |
| #マーケティング | マーケティング関連 |
| #開発 | 開発・技術関連 |
| #重要 | 優先度高・要確認 |
| #アーカイブ | 完了・保存のみ |

---

## 6. エージェント別 主な操作フロー

### 秘書エージェント
- ミーティング後 → `ideas/` または `projects/` に要約ノートを作成
- 週次レポート → 各フォルダのノートを参照してサマリー作成

### 営業エージェント
- 新規見込み客 → `clients/[企業名]/` フォルダを作成し overview を記録
- 商談後 → クライアントノートに商談履歴を追記
- リサーチ結果 → `research/` にノートを保存

### PMエージェント
- 新規プロジェクト → `projects/[名前]/` フォルダを作成
- 進捗更新 → `projects/[名前]/progress.md` を更新

### 開発エージェント
- 納品物 → `projects/[名前]/deliverables.md` に記録
- 技術メモ → `projects/[名前]/technical_notes.md` に保存

### マーケティングエージェント
- 市場調査 → `research/` に保存
- コンテンツアイデア → `ideas/` に保存
