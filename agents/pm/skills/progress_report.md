## 概要
並走中の複数タスクの進捗を各エージェントから収集し、全体状況をCEOが一目で把握できるレポートにまとめる。
ボトルネック（待機中・決定が必要なタスク）を明確にし、CEOの判断が必要な事項を優先して提示する。

## 入力
- 各エージェントからの進捗報告（直接受領またはmemory.md・logsから収集）
- 現在並走中のタスクリスト

## 実行手順
1. **進捗情報の収集**：秘書（菅）経由または直接 `[MSG:チャンネル名]` で各エージェントに確認する
2. **ステータス分類**：各タスクを以下に分類する
   - 完了（Done）
   - 進行中（In Progress）
   - 待機中・ブロック（Blocked）
   - 未着手（Not Started）
3. **ボトルネック特定**：「誰かの完了を待っている」または「CEOの決定が必要」なタスクを抽出する
4. **サマリー作成**：Discord向けの簡潔な進捗サマリーを作成する（タスクごとに1〜2行）
5. **意思決定事項の提示**：CEOに判断を求める事項（例：「どのセグメントから優先攻略するか」）を明示する
6. **次のアクションの提案**：ブロック解消のための具体的な提案を添える

## 出力
- 進捗サマリーレポート（Discord用Markdownフォーマット）
- CEO向けの意思決定リスト（箇条書き）
- 保存先：`logs/YYYY-MM-DD_グレン-pm.md` に報告内容を記録

## 出力フォーマット例（Discord用）
```
🗂️ **プロジェクト進捗レポート**
━━━━━━━━━━━━━━━━
✅ 完了：営業リストv3（20社）作成 - グラント
🔄 進行中：セグメント分析 - 森岡（着手済み）
⏳ 待機中：価格設計試算 - ウォズニアック（グレン確認待ち）
⏳ 待機中：LPモックアップ - ウォズニアック（クライアント情報待ち）

⚠️ CEO判断が必要：飲食/フィットネス/小売のどのセグメントから優先攻略するか
━━━━━━━━━━━━━━━━
```
