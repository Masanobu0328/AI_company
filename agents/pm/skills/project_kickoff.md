## 概要
CEOから事業方針やプロジェクト開始の指示を受けた際に、全体の作業フローを整理し、各エージェントへのタスクを分解・割り振る。
依存関係（どのタスクが完了してから次に進めるか）を明確にして並走できるタスクは並走させ、ボトルネックを事前に特定する。

## 入力
- CEOからのプロジェクト概要・目的・期待する成果物
- 関与するエージェントの一覧と各自の役割・現在の稼働状況

## 実行手順
1. **プロジェクト概要の確認**：CEOの指示を整理し、目的・対象・期待成果物を1段落でまとめる
2. **タスク分解（WBS）**：プロジェクトをフェーズ・タスク・サブタスクに分解する
3. **依存関係の整理**：どのタスクが他タスクの完了を待つ必要があるか（例：セグメント分析完了→アウトリーチ文章作成）を明示する
4. **エージェントへの割り振り**：各タスクを担当エージェントに割り当てる
   - 営業リスト作成・アウトリーチ → グラント（営業）
   - セグメント分析・SNS投稿 → 森岡（マーケティング）
   - 価格設計・LP制作 → ウォズニアック（開発）
   - 全体進捗管理・CEO報告 → グレン（PM）
5. **指示メッセージの送信**：各エージェントへ `[MSG:チャンネル名: 依頼内容]` 形式で指示を送る
6. **秘書への連絡**：進捗フローの開始を菅（秘書）に共有し、進捗収集スケジュールを調整する

## 出力
- タスク割り振り表（Markdown形式）
- 各エージェントへの指示メッセージ（Discord用）
- 保存先：`logs/YYYY-MM-DD_グレン-pm.md` にキックオフ内容を記録
