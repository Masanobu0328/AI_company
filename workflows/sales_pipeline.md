# 営業パイプライン - ワークフロー定義

## 概要
見込み客の発掘から成約・引き継ぎまでの全プロセスを定義する。

## ステージ定義

### ステージ1: 見込み客調査（Prospecting）
**担当**: 営業エージェント
**インプット**: ターゲット業界・条件
**アウトプット**: 見込み客リスト

タスク:
- [ ] ターゲット業界の定義
- [ ] 企業リストの作成（企業名・URL・担当者）
- [ ] 初期スクリーニング（適合度評価）

---

### ステージ2: Webサイト分析（Website Analysis）
**担当**: 営業エージェント
**インプット**: 企業URL
**アウトプット**: 分析レポート

タスク:
- [ ] デザイン品質の評価（1〜10点）
- [ ] UX問題点のリストアップ
- [ ] 改善提案の作成
- [ ] 競合比較

---

### ステージ3: モックアップ依頼（Mock Request）
**担当**: 営業 → 開発エージェント
**インプット**: 分析レポート + 企業情報
**アウトプット**: Webサイトモックアップ

タスク:
- [ ] 開発エージェントへのブリーフ送付
- [ ] モックアップ受領・確認
- [ ] 提案書への組み込み

---

### ステージ4: アウトリーチ（Outreach）
**担当**: 営業エージェント
**インプット**: 分析レポート + モックアップ
**アウトプット**: 送信済みメッセージ

タスク:
- [ ] パーソナライズドメッセージ作成
- [ ] メール/DM送信
- [ ] フォローアップスケジュール設定

---

### ステージ5: ディスカバリー（Discovery）
**担当**: 営業エージェント + CEO
**インプット**: クライアントとのミーティング
**アウトプット**: ディスカバリーメモ

タスク:
- [ ] ニーズヒアリング
- [ ] 予算・期限の確認
- [ ] 技術要件の把握
- [ ] ミーティング要約（Secretary担当）

---

### ステージ6: プロジェクトブリーフ（Project Brief）
**担当**: 営業エージェント
**インプット**: ディスカバリーメモ
**アウトプット**: プロジェクトブリーフ書

タスク:
- [ ] 要件定義書の作成
- [ ] 見積もりの作成
- [ ] クライアント承認取得

---

### ステージ7: 開発引き継ぎ（Handoff to Development）
**担当**: 営業 → PM → 開発エージェント
**インプット**: 承認済みプロジェクトブリーフ
**アウトプット**: 開発スタート確認

タスク:
- [ ] PMへのプロジェクト登録
- [ ] 開発エージェントへのブリーフ共有
- [ ] キックオフミーティング設定

## ステータス管理

| 📊 分析中 | Webサイト分析段階 |

| 📝 ブリーフ作成中 | 要件定義中 |
| 🚀 開発中 | 開発チームに引き継ぎ済 |
| ✅ 成約 | 契約完了 |
