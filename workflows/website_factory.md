# Webサイトファクトリー - ワークフロー定義

## 概要
クライアントのWebサイト制作を効率的に量産するためのワークフロー。

## コンセプト
「先にモックアップを見せる」バリューファースト戦略により、
クライアントが価値を実感してから契約するモデル。

## フロー図
```
[営業] 見込み客特定
    ↓
[営業 → 開発] モックアップ依頼
    ↓
[開発] モックアップ生成（24時間以内）
    ↓
[営業] クライアントへ提案
    ↓
[クライアント] 承認 / 修正依頼 / 不採用
    ↓（承認の場合）
[開発] 本番サイト構築
    ↓
[開発 → 営業 → クライアント] 納品
```

## モックアップ依頼テンプレート

```
## モックアップ依頼

**企業名**: [企業名]
**業界**: [業界]
**現行サイトURL**: [URL]
**分析結果**:
- デザイン問題: [内容]
- UX問題: [内容]
- 改善ポイント: [内容]

**要望**:
- スタイル: [モダン/クラシック/ミニマル等]
- カラー: [ブランドカラーがあれば]
- 参考サイト: [URL]

**期限**: [日付]
```

## 開発スタンダード

### 使用技術
- HTML5 / CSS3 / Vanilla JS（標準）
- Tailwind CSS（スタイリング）
- 必要に応じてReact/Next.js

### ファイル構成
```
clients/[クライアント名]/
├── mock/          # モックアップファイル
│   ├── index.html
│   └── style.css
├── final/         # 最終納品ファイル
└── brief.md       # プロジェクトブリーフ
```

### 品質チェックリスト
- [ ] レスポンシブ対応（モバイル・タブレット・PC）
- [ ] クロスブラウザ確認（Chrome/Firefox/Safari/Edge）
- [ ] 表示速度テスト（Lighthouse 80点以上）
- [ ] アクセシビリティ確認
- [ ] リンク動作確認

## SLA（サービスレベル合意）
| タスク | 目標時間 |
|---|---|
| モックアップ生成 | 24時間以内 |
| フィードバック反映 | 12時間以内 |
| 本番サイト構築 | 3〜5営業日 |
| 緊急対応 | 4時間以内 |
