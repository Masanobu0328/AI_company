# ナレッジベース

## 概要
Obsidian Vault（Google Drive同期）で管理する会社の知識資産。
全エージェントがMarkdownファイルを読み書きする中央ストレージ。

## フォルダ構造
```
knowledge_base/
├── README.md              # このファイル
├── agents_kb_guide.md     # エージェント操作ガイド
├── _templates/            # ノートテンプレート集
├── ideas/                 # アイデア・着想メモ
├── research/              # リサーチ・業界調査
├── clients/               # クライアント情報
└── projects/              # 進行中・完了プロジェクト
```

## フォルダ別用途
| フォルダ | 担当エージェント | 用途 |
|---|---|---|
| ideas/ | 全エージェント | 自由なアイデア・着想の記録 |
| research/ | 営業・マーケティング | 市場調査・競合・業界データ |
| clients/ | 営業 | クライアント情報・商談履歴 |
| projects/ | PM・開発 | プロジェクト管理・進捗 |

## ファイル命名規則
```
# 通常ノート
YYYY-MM-DD_[タイトル].md

# クライアントノート
clients/[企業名]/[企業名]_overview.md

# プロジェクトノート
projects/[プロジェクト名]/brief.md
```

## Obsidianリンク記法
- ノート間リンク: [[ノート名]]
- 見出しリンク: [[ノート名#見出し]]
- タグ: #クライアント #リサーチ #アイデア

## エージェント操作ルール
1. ノートはMarkdown形式で保存する
2. ファイル先頭にFrontmatterを必ず記述する
3. 関連ノートは [[]] リンクで相互参照する
4. タグを付けて検索可能にする
5. 更新時は `更新日` フィールドを書き換える
