| #一般 | 自由利用 |

## メモリ構造（3層）

### Layer 1：コアメモリ（常時ロード・小さく保つ）
- `agents/[名前]/memory.md` → エージェント個人の記憶・CEOからのフィードバック
- `knowledge/shared/` → 会社全体の方針・ルール（全エージェント共通）
- 目標トークン数：合計500〜800トークン以内

### Layer 2：直近の会話（スライディングウィンドウ）
- チャンネルごとに直近10ターンをメモリ内保持
- Botが再起動するとリセット

### Layer 3：全会話ログ（保存のみ・APIには送らない）
- `logs/YYYY-MM-DD_[チャンネル名].md` に自動保存
- 毎日23:59にまとめてGitHubにpush
- 将来的にRAG（検索型）対応予定

## 学習・記憶の仕組み

### 個別チャンネルで「覚えて」と言った場合
→ `agents/[エージェント名]/memory.md` に追記
→ 即座にエージェントのプロンプトに反映（再起動不要）

### #agent-chat で「覚えて」と言った場合
→ `knowledge/shared/YYYY-MM.md` に追記
→ 全エージェントが次回起動時から参照

## 自動スケジュール

| 時刻（JST） | 処理 |
|---|---|
| 毎朝 09:00 | PMが朝会を開始 → 全員が #agent-chat で発言 → GitHub保存 |
| 毎朝 09:00 | 森岡毅がThreads投稿案3本を #task-board に提出 |
| 毎日 23:59 | 当日の会話ログをまとめてGitHubにpush |

---

## PMのタスク委譲フロー

```
CEOが #グレン-PM にタスクを投げる
→ PMが返答 + [→dev: ...] [→sales: ...] を自動生成
→ #agent-chat に委譲タスクが投稿される
→ 各エージェントが自動応答
→ #task-board に進捗・完了が通知される
```
