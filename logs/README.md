# ログシステム

## 概要
全エージェントの活動ログをGitHubリポジトリで管理する。
すべてのログは日本語で記述する。

## ディレクトリ構造
```
logs/
├── README.md              # このファイル（ログシステム定義）
├── sessions/              # カテゴリ別ログファイル
│   ├── meetings/          # ミーティング・会議ログ
│   ├── sales/             # 営業活動ログ
│   ├── marketing/         # マーケティングログ
│   ├── development/       # 開発作業ログ
│   └── system/            # システム・管理ログ
├── daily/                 # 日次サマリー（Secretary作成）
└── incidents/             # インシデントログ
```

## カテゴリ説明
| カテゴリ | 担当エージェント | 記録内容 |
|---|---|---|
| meetings | Secretary | ミーティング議事録・要約 |
| sales | 営業 | アウトリーチ・商談・成約記録 |
| marketing | マーケティング | コンテンツ・キャンペーン記録 |
| development | 開発 | 制作・実装・納品記録 |
| system | PM / Secretary | システム操作・設定変更記録 |

## ファイル命名規則
```
YYYY-MM-DD_[種別]_[概要].md
例:
  2026-03-11_meeting_週次定例.md
  2026-03-11_sales_株式会社ABC_アウトリーチ.md
  2026-03-11_dev_モックアップ_株式会社XYZ.md
```

## ログ記録ルール
1. すべての重要なアクションはログに記録する
2. ログは必ず日本語で記述する
3. エラー・問題発生時は `incidents/` にも記録する
4. 日次サマリーはSecretaryが毎日EODに作成する
5. ファイル名は命名規則に従う
